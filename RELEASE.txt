Version 0.1.2 (2026-07-07)
---------------------------
More style refinements from running the CLI on real topics.
- Shared LEDGER state: scoring in `## Ранжування` (Суддя rates each hypothesis ★1-5 on 4 axes; explore unsourced, investigate sourced) and arguments in `## Аргументи` (Дослідник/Опонент write за/проти per hypothesis with sources -- a debate is now skimmable, not a blob).
- Weave renders both from the ledger; `open` shows scores and a за/проти tally; a rebuild never drops an idea or its arguments.
- DOCUMENT.md is written for a reader NEW to the topic (explanatory, not telegraphic); no single winner in any style.
- explore is divergent (Ф->Ф->А->Ф->С->Т, score-only evaluate).
- run --target accepts a comma list -> runs the style once per hypothesis.
- Live per-move progress to stdout + progress.log; engine.log keeps one line per run (styles executed) with time / tokens / cost from the claude envelope.
- MURARI_RUN_TIMEOUT (default 900s / 15 min) tunes the per-move timeout; a failed move rolls back only itself, keeping completed (paid) moves.
- Roadmap: session auto-naming via Haiku queued as a v0.2 task.

Version 0.1.1 (2026-07-06)
---------------------------
Style behaviour refinements (found while running the CLI on real topics).
- explore is now divergent (Ф->Ф->А->Ф->А->Т): breadth via generate+mutate, no evaluate/deepen, no verdict.
- Style-shaped moves: wilder Фантазер in explore/riff; Дослідник gathers evidence both for and against (no verdict); Ткач catalogs (no winner) in explore/debate, state-of-analysis synthesis in the convergent styles.
- Every DOCUMENT.md closes with a multi-axis ranking table of all hypotheses (★1-5: доказовість / оригінальність / популярність / пояснювальна сила) -- a scorecard, not a single winner.
- run --target Hxx pins deepen/oppose/mutate to a chosen hypothesis; open lists the H-ids.
- Product framing: murari is an environment for analysing ideas, not a machine that picks a winner -- verdicts prune the disproven, the human decides.

Version 0.1.0 (2026-07-05)
---------------------------
Phase v0.1 -- Orchestration (headless). The by-hand agent of v0.0 is now driven by a Python orchestrator with a headless CLI.
- Project scaffolding (MUR-006): pyproject (ruff + pytest), the murari package, config and session budgets (MURARI_RUNS/MAX_TURNS/MODEL/HOME).
- Agent canon v2 + contract v2 (MUR-007): role-parameterized canon installed at .claude/agents/brainstormer.md; the JSON output contract with the source gate and a fence/preamble-tolerant extractor.
- LEDGER v2 read side (MUR-008): parser for H-ids, statuses, lineage (parents/mutation), the run journal, and the dry-run counter; per-move productivity check.
- AgentRunner seam (MUR-009): the verified `claude -p` invocation (canon via --append-system-prompt, per-role tool narrowing, Bash/Task disallowed); MockAgentRunner for zero-cost tests.
- Session lifecycle (MUR-010): create / open-and-continue / list, with snapshot-restore so a failed run leaves the workspace byte-identical.
- Style engine + CLI (MUR-011): six styles (explore/debate/riff/investigate(default)/evolve/premortem) run move by move, seeded mutation/combine choices, budgets, the deviation rule after two dry moves, and the DOCUMENT ownership guard (only weave writes it). Headless CLI: new / open / run / list.
- Runnable end-to-end: Opus runs on the MAX subscription (API key stripped from the subprocess), `python -m murari` entrypoint, legible nonzero-exit diagnostics, and a nullable contract `basis`.
- 122 offline tests; CI mocks the agent, Haiku, and web search -- no paid APIs.
